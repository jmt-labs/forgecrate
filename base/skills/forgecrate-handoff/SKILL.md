# Handoff

Generiert eine `HANDOFF.md` mit portablem Projekt-Kontext für AI-Modellwechsel oder Session-Übergabe. Kein externes Tool nötig.

## Ablauf

**Schritt 1 — Daten sammeln (parallel ausführen):**

```bash
# Git-Info
git branch --show-current && git log --oneline -10 && git status --short
date "+%Y-%m-%d %H:%M:%S"

# Stack erkennen (erstes Match)
for f in package.json Cargo.toml pyproject.toml go.mod; do
  [ -f "$f" ] && cat "$f" && break
done

# Verzeichnisstruktur (2 Ebenen, Noise ausblenden)
find . -maxdepth 2 \( -name .git -o -name node_modules -o -name .claude \
  -o -name target -o -name __pycache__ -o -name dist -o -name .venv \
  -o -name .next -o -name build -o -name vendor -o -name coverage \) \
  -prune -o -print | sort | head -60

# TODOs und FIXMEs
grep -rEn '\b(TODO|FIXME)[:([ ]' \
  --include="*.go" --include="*.ts" --include="*.tsx" \
  --include="*.py" --include="*.rs" --include="*.js" \
  . 2>/dev/null | grep -v node_modules | grep -v ".git" | head -20
```

`CLAUDE.md` lesen falls vorhanden (für Conventions).

**Schritt 2 — HANDOFF.md schreiben:**

```
# HANDOFF — <Projektname>
> Generated by `/forgecrate-handoff` at <Zeitstempel>
> Branch: `<branch>` | Last commit: <hash> <message>

## Stack
<Sprache + Framework | Package Manager>
<ORM: X | DB: X | Auth: X | Deploy: X | Test: X — nur was erkannt wurde>

## Structure
<Verzeichnisbaum 2 Ebenen, kurze Inline-Kommentare für nicht-offensichtliche Dirs>

## Conventions
<Bullet-Liste aus File-Patterns, Configs, CLAUDE.md, Linter-Configs>

## Recent Activity
<letzte 5–10 Commits: `<hash>` <message> (<relative Zeit>)>

## Current State
<uncommitted changes mit M/A/D-Prefix pro Datei, oder "Working tree clean">

## Known Issues
<TODO/FIXME mit file:line — Abschnitt weglassen wenn keine gefunden>
```

**Schritt 3 — Abschluss:**

Datei nach `HANDOFF.md` im aktuellen Verzeichnis schreiben und dem User Pfad und Dateigröße bestätigen.
