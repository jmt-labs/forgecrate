Generate a HANDOFF document for the current repository and write it to `HANDOFF.md`. Do NOT install any tools.

**Step 1 — Gather data via Bash (run in parallel where possible):**

- `git branch --show-current && git log --oneline -10 && git status --short`
- `date "+%Y-%m-%d %H:%M:%S"`
- `cat package.json 2>/dev/null || cat Cargo.toml 2>/dev/null || cat pyproject.toml 2>/dev/null || cat go.mod 2>/dev/null` (first match only)
- `find . -maxdepth 2 -not -path './.git/*' -not -path './node_modules/*' -not -path './.claude/*' -not -path './target/*' -not -path './__pycache__/*' | sort | head -60`
- `grep -rn "TODO\|FIXME" --include="*.go" --include="*.ts" --include="*.tsx" --include="*.py" --include="*.rs" --include="*.js" . 2>/dev/null | grep -v node_modules | grep -v ".git" | head -20`
- Read `CLAUDE.md` if present for conventions

**Step 2 — Render in this exact format and write to `HANDOFF.md`:**

```
# HANDOFF — <project name>
> Generated by `handoff` at <timestamp>
> Branch: `<branch>` | Last commit: <hash> <message>

## Stack
<language + framework | package manager>
<ORM: X | DB: X | Auth: X | Deploy: X | Test: X — only what's detected>

## Structure
<directory tree top 2 levels, brief inline comment for non-obvious dirs>

## Conventions
<bullet list inferred from file patterns, configs, CLAUDE.md, linter configs>

## Recent Activity
<last 5–10 commits: `<hash>` <message> (<relative time>)>

## Current State
<uncommitted changes with M/A/D prefix per file, or "Working tree clean">

## Known Issues
<TODO/FIXME with file:line — omit entire section if none found>
```

**Step 3 — After writing `HANDOFF.md`:** confirm the file path and token count to the user.
